## One does not simply mark all messages as read

A small personal data project about navigating multi-channel communication.

### What this is

This project explores how messages from different sources (Telegram channels and email) contribute to information overload and the subjective feeling of “too much pressure” in everyday digital communication.

### Data

The dataset consists of message exports from three two Telegram channel and a email newsletter from the same educational project Simulative. 

Time coverage from September 2025 to April 2026  

### What is included

For each message, the following signals are analyzed:

- message volume over time
- message length
- duplication across channels
- content type and structure
- stylistic features
- degree of text transformation (including possible AI-assisted rewriting)
- inferred metadata such as author and tags

Non-text messages (images, voice notes, attachments) are not analyzed directly, but are used for estimating overall message volume and notification load.

### Goal

The goal is not to build a predictive system, but to understand:

- how overlapping communication channels shape perception
- when and why content starts to feel “too intense”
- how duplication and message density contribute to disengagement

**Or in simpler terms: when “being informed” starts to feel like “too much”.**
